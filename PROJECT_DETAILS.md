Бұл кеңейтілген құжат **Library System** платформасының техникалық құрылғыларын, модульдік жүйесін және пайдаланушы процестерін сипаттайды. Қосымшаның мақсаты — кітаптар қорын цифрландыру және оқырмандармен өзара әрекеттесуді оңтайландыру.

## Жүйенің құрылымы (Core Features)

| Модуль | Компонент / Файл | Сипаттамасы мен Логикасы |
| :--- | :--- | :--- |
| **Авторизация (Auth)** | `app/api/auth/[...nextauth]` | `NextAuth.js` кітапханасы негізінде жасалған басқару (authorization). `bcryptjs` арқылы жоғары деңгейлі құпия сөз қорғанысы орнатылған. |
| **Пайдаланушы (Client)** | `app/(client)` | Тұтынушыға арналған ортака (UI). Негізгі бетте танымал кітаптар, соңғы қосылған әдебиеттер мен санаттық сүзгілер ұсынылады. |
| **Кітап алу (Borrowing)** | `components/BorrowButton.tsx` | Кітапты жалға алу транзакциясы. Жүйе әрбір кітаптың қоймадағы санын (`total_copies` - `borrowed_count`) нақты уақытта есептейді. |
| **Әкімші тақтасы (Admin)** | `app/admin/dashboard` | Тек жоғары құқығы бар (role: `admin`) пайдаланушыларға рұқсат етілген. Оқырмандар мен кітаптардың егжей-тегжейлі кестелік (Table) көрінісі. |
| **API Эндпойнттар** | `app/api/books`, `app/api/borrow` | Клиент-сервер арасындағы тұрақты мәлімет алмасу логикасы: жаңа кітап қосу, оқырманға бекіту, қайтару мерзімін орнату. |

## Базалық API Бағыттары (REST Endpoints)

Барлық REST сұраныстары `Next.js App Router` маршруттарында орналасқан және сессия арқылы қорғалған.

| Route | Әдіс (Method) | Мақсаты | Қатынас деңгейі (Role) |
| :--- | :--- | :--- | :--- |
| `/api/books` | `GET` | Бүкіл кітаптар каталогын немесе іздеу нәтижесін алу | Барлық пайдаланушылар |
| `/api/admin/books` | `POST` | Жаңа кітапты жүйеге енгізу, қойма санын орнату | Әкімші (Admin) ғана |
| `/api/admin/books/[id]`| `PATCH`, `DELETE`| Кітап мағлұматтарын жаңарту (атауы, авторы) немесе жою | Әкімші (Admin) ғана |
| `/api/borrow` | `POST` | Кітапты оқуға алу (Borrow). Дерекқордағы дана санын азайту. | Авторизацияланған оқырман |
| `/api/borrow/[id]` | `PATCH` | Кітапқа қайтарылған (Returned) статусын беру. | Авторизацияланған оқырман / Әкімші |

## Мәліметтер Базасының (DB) логикасы

Платформада **Neo Postgres** бұлтты жүйесі қолданылады (Drizzle ORM арқылы). Ең күрделі процесс `Borrowing` (Жалға алу) есебінде орналасқан:
1. Оқырман `Borrow` батырмасын басқанда, сервер баптауында кітаптың `available_copies` бағаны тексеріледі.
2. Егер санды мән `> 0` болса, `borrowings` кестесіне жаңа жазба түседі (`status: borrowed`, `due_date: +14 күн`).
3. Сол уақытта `books` кестесінде қолжетімді кітаптар саны азаяды.
4. Қайтару (`Return`) рәсімі барысында жоғарыда аталған қадамдар қарама-қарсы (reverse) бағытта жүзеге асады.

## Дизайн Және Тестілеу

Бұл қосымша максималды түсінікті «баспагерлік-минималистік» визуалмен жарақталған:
- **Мәтін оңай оқылады:** Ашық фонда (light mode) өткір контраст пен заманауи геометрия ұсынылған. Тегін `Tailwind` болмағандықтан, таза `CSS` қолданылған.
- **Жоғары өнімді сапа:** Әр компонент SSR (Server-Side Rendering) қабілетін сақтап, іске қосылу (load) жылдамдығын қамтамасыз етеді. 

Бұл архитектура үлкен деректер көлемінде де (мыңдаған кітаптар мен пайдаланушылар) мүлтіксіз жұмыс істеуге толықтай дайын.
